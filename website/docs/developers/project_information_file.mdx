# Tansu information file

Metadata on the projects are defined in a TOML file using the [SEP-1](https://github.com/stellar/stellar-protocol/blob/master/ecosystem/sep-0001.md) specification-which standardize the Stellar Info File. This file is to be named `tansu.toml` as to avoid the confusion and collision with `stellar.toml`.

- `DOCUMENTATION`: information about the project
- `ACCOUNTS`: public keys of the maintainers
- `PRINCIPALS`: information about maintainers. For hosted git providers this is typically a GitHub-style handle; for public Radicle repositories this can be a Radicle alias.

For software projects, repository metadata can be represented in two ways:

- Hosted git providers such as GitHub, GitLab, Bitbucket, Codeberg, and Gitea use `ORG_GITHUB` to store the repository path while the canonical repository URL is stored on-chain.
- Public Radicle repositories use the canonical `rad:<RID>` repository identifier on-chain and may include optional metadata such as `ORG_REPOSITORY_PROVIDER="radicle"` and `ORG_REPOSITORY_SEED="seed.radicle.xyz"` in `tansu.toml` to preserve the preferred public seed for browsing.

## Display Name (ORG_DBA)

When registering a project, you specify a **project name** (e.g., `myproject`) which is the on-chain identifier limited to 15 lowercase letters.

The `ORG_DBA` field (Doing Business As) in your `tansu.toml` allows you to specify a **display name** that can be different from your project name. This is useful for:

- Organizations that want a more descriptive project name
- Projects with names longer than 15 characters

:::note[Order matters]

The order of both `ACCOUNTS` and `PRINCIPALS` is important as to link a maintainer with its public key.

:::
