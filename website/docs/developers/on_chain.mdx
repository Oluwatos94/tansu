# On-Chain

The Stellar smart contract can be directly used and inspected either using an explorer or via the convenient [Stellar CLI](https://github.com/stellar/stellar-cli). See the [contract address](./dapps_and_contracts.mdx).

## Typical flow

The first step is to register a project by providing some information. This will notably check that the maintainer doing this action owns a [Soroban Domain](https://sorobandomains.org/) for the project's name. If that's not the case, it will attempt to register one for a year.

```bash
stellar contract invoke \
	--source-account G... \
	--network testnet \
	--id C... \
	-- \
	register \
	--maintainer G... \
	--name tansu \
	--maintainers '[G...,G...]' \
	--url https://github.com/your_project \
	--ipfs QmYourTansuTomlCID...
```

![Contract on-chain](/assets/docs/contract.png)

Then maintainers can commit changes on-chain as they push new commits.

```bash
stellar contract invoke \
    --source-account G... \
    --network testnet \
    --id C... \
    -- \
    commit \
    --maintainer G... \
    --project_key ... \
    --hash ...
```

![Contract on-chain](/assets/docs/storage.png)

![transactions](/assets/docs/transactions.png)

Maintainers can also attach generic supply-chain evidence to a specific commit. Evidence is keyed by project, commit hash, and kind (`Sbom`, `Cve`, or `Attestation`). The contract stores only an off-chain content identifier; it does not upload artifacts, parse evidence contents, or verify attestations on-chain. See [Commit Evidence](./evidence.mdx) for the automated producer path (generate artifact → upload to IPFS → record CID).

```bash
stellar contract invoke \
    --source-account G... \
    --network testnet \
    --id C... \
    -- \
    set_evidence \
    --maintainer G... \
    --project_key ... \
    --commit_hash ... \
    --kind Sbom \
    --cid bafy...
```

Later on, anyone can fetch on-chain data about a project and verify the last commit hash for themselves.

```bash
stellar contract invoke \
    --source-account G... \
    --network testnet \
    --id C... \
    -- \
    get_commit \
    --project_key ...
```

Anyone can also fetch evidence for a specific commit and evidence kind.

```bash
stellar contract invoke \
    --source-account G... \
    --network testnet \
    --id C... \
    -- \
    get_evidence \
    --project_key ... \
    --commit_hash ... \
    --kind Sbom
```

## Events

Registering a project, committing a new hash, or setting commit evidence sends a contract event.

![Contract events off-chain](/assets/docs/events.png)

These events can be observed directly from the chain. The dApp can also call the contract read methods and fetch evidence artifacts from IPFS by CID, without a backend or database.

## Git Hooks

A pre-push hook is available to ensure commits are recorded on-chain before pushing:

```bash
pre-commit install -t pre-push
```
